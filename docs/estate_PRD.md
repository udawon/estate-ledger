# Estate-Ledger — 제품 요구사항 정의서 (PRD)

> **버전**: v1.0.0
> **작성일**: 2026-03-03
> **상태**: 진행 중 (MVP)

---

## 1. 제품 개요

### 1.1 제품명
**Estate-Ledger** — 부동산 통합관리 시스템

### 1.2 한 줄 설명
부동산 중개사가 매매·상가·전월세 매물을 한 곳에서 관리하고, AI 엔진으로 입지를 자동 분석하는 통합 업무 도구

### 1.3 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 분산된 매물 관리 | 엑셀·메모·앱이 뒤섞여 매물 현황 파악이 어려움 |
| 입지 분석의 비효율 | 교통·상권·환경·안전 데이터를 수작업으로 취합하는데 수 시간 소요 |
| 부정확한 정보 | 실거래가·공급량·환경 위험 등을 빠르게 검증할 수단 없음 |
| 데이터 고립 | Streamlit 기반 도구는 팀 공유·모바일 접근이 어려움 |

### 1.4 해결 방법

Next.js 기반 **단일 웹 앱**으로 매물 장부(매매·상가·전월세)와 **7카테고리 AI 입지분석 엔진**을 통합한다.
주소 하나만 입력하면 30초 안에 교통·일자리·생활·교육·환경·미래가치·공급 점수와 등급(A~F)을 제공하며,
매물 선택 → 분석 연동으로 현장에서 바로 활용 가능한 인사이트를 제공한다.

## 2. 목표 및 성공 지표

### 2.1 비즈니스 목표 (MVP)
- 매물 등록·수정·삭제·검색 전 기능을 웹에서 사용 가능하도록 전환
- 입지분석 결과의 정확도 향상 (서울 25개 구 + 경기·인천 25개 시·구 지원)
- Streamlit 앱 완전 대체 — 설치 없이 브라우저에서 바로 사용

### 2.2 성공 지표

| 지표 | 기준 |
|------|------|
| 매물 등록 소요 시간 | 1건 3분 이내 |
| 입지분석 응답 시간 | 30초 이내 (Kakao 연동 시) |
| Excel 내보내기 성공률 | 100% (데이터 손실 없음) |
| 로그인 → 첫 화면 진입 | 3초 이내 |

| 구분 | 설명 |

### 3.2 페르소나

**관리자 — "민준"** (공인중개사, 사무소 운영 3년차)
- 매일 매물 현황을 한눈에 확인하고 싶음
- 고객에게 현장에서 바로 입지 정보를 제공하고 싶음
- 엑셀 파일로 데이터 백업·공유가 필수

**데모 사용자 — "소연"** (부동산 투자 관심 직장인)
- 관심 주소의 입지 점수를 빠르게 확인하고 싶음
- 교통·학군·환경 등 구체적인 근거가 필요함
- 회원가입 없이 서비스를 체험해보고 싶음

## 11. 용어 정의

| 용어 | 설명 |
|------|------|
| **Admin** | 관리자 계정. 매물 CRUD 및 비밀번호 변경 권한 보유 |
| **Demo** | 데모 체험 계정. 매물 조회·분석 가능, 비밀번호 변경 불가 |
| **매물장** | 매매·상가·전월세 매물을 관리하는 테이블 화면 |
| **입지분석** | 주소 기반으로 교통·생활·환경 등 7개 카테고리를 평가하는 AI 엔진 |
| **totalScore** | 7카테고리 가중 평균 점수 (패널티 적용 전) |
| **finalScore** | 패널티를 적용한 최종 점수 (등급 산출 기준) |
| **패널티** | 혐오시설·나들목 인접·침수구역 등에 부과되는 감점 (최대 -30pt) |
| **Grade** | 등급. A(85+) / B(70+) / C(55+) / D(40+) / F(40 미만) |
| **district** | 시·구 단위 행정구역 (예: 강남구, 광명시). 정적 데이터 조회 기준 |
| **ce_session** | JWT 세션 쿠키 이름 (httpOnly, `SameSite=Lax`) |
| **SQLite DB** | `data/estate.db` — better-sqlite3로 관리하는 로컬 파일 DB |
