# landing-page-improvement Plan

> **목표**: 랜딩 페이지 전환율 및 서비스 신뢰도 향상
> **작성일**: 2026-03-02
> **우선순위**: Medium

---

## 1. 현황 분석

### 1.1 현재 구성
| 섹션 | 현황 | 문제점 |
|------|------|--------|
| Hero | 주소 입력 폼 + 4개 뱃지 | 실제 7개 카테고리인데 4개만 표시, 실측 강점 미표현 |
| Features | 4개 카드 (교통/상권/환경/안전) | 7개 카테고리와 불일치, 설명이 실제 기능과 다름 |
| HowItWorks | 4단계 스텝 | 내용은 적절하나 "보고서" 스텝이 실제와 다름 |
| CTA | 무료 분석 버튼 | 사회적 증거·신뢰 지표 없음 |

### 1.2 핵심 개선 기회
1. **카테고리 불일치**: Features가 4개이나 실제 분석은 7개 카테고리
2. **신뢰 지표 부재**: 공공 API 5개 실시간 연동 강점이 전혀 드러나지 않음
3. **사회적 증거 없음**: 수치("30초", "7개 지표" 등) 부각 필요
4. **텍스트 정확도**: "상권", "환경", "안전" → 실제명: "생활인프라", "환경위험" 등

## 3. 구현 계획

### 3.1 컴포넌트별 변경

- 서브타이틀: "공공 API 5개 실시간 연동" 신뢰 문구 추가
